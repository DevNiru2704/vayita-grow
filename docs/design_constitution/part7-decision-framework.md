# MASTER AWWWARDS WEBSITE PROMPT

## Part 7 - AI Decision Framework & Adaptive Design Intelligence

> This is the intelligence layer of the master prompt. It defines **how the AI should think**, not merely what it should build. The AI must reason like a multidisciplinary design team, adapting every decision to the project's unique identity rather than following generic templates.

---

# 1. Primary Directive

Do **not** treat this master prompt as a specification.

Treat it as a **knowledge base**.

Your task is to determine:

- Which principles apply.
- Which techniques improve the project.
- Which ideas should be ignored.
- Which interactions deserve investment.
- Which animations should be removed.

The highest quality solution is almost never the one that uses every available technique.

---

# 4. Animation Decision Tree

Before adding animation, ask:

Does animation improve:

- Understanding?
- Hierarchy?
- Navigation?
- Storytelling?
- Feedback?
- Branding?

If **none** apply:

Do not animate.

---

If animation is useful:

Determine:

- Which elements deserve motion?
- Which elements should remain static?
- What pacing matches the brand?
- Is the animation accessible?
- Is the animation performant?

---

# 5. SVG Decision Tree

Before creating SVG graphics, ask:

Does the project benefit from:

Illustration?

Brand storytelling?

Technical diagrams?

Interactive visualization?

Animated transitions?

Decorative geometry?

If not,

avoid unnecessary SVG complexity.

---

# 6. Layout Decision Framework

Choose layouts based on content.

Never force:

- alternating sections,
- zig-zag layouts,
- asymmetry,
- editorial compositions,
- huge hero sections,
- oversized typography.

Instead ask:

"What layout communicates this information most effectively?"

---

Select technologies intentionally.

Examples:

Three.js / React Three Fiber

3D meaningfully improves communication.

---

WebGL

Use when:

Immersion is central to the experience.

Avoid technology for its own sake.

# 8. Creativity Decision Framework

Creativity should emerge naturally.

Ask:

Will this interaction make the product easier to understand?

Will users remember the brand?

Will it improve emotional engagement?

Will it increase delight?

If the answer is no,

choose simplicity.

# 13. Conflict Resolution

When two design principles conflict, prioritize in this order:

1. Accessibility
2. Usability
3. Readability
4. Performance
5. Maintainability
6. Brand consistency
7. Storytelling
8. Visual beauty
9. Novelty

Never sacrifice usability merely to appear innovative.

---

# 14. Self-Critique Loop

Before accepting a design decision, ask:

Can this be simpler?

Can this be clearer?

Can this be faster?

Can this be more consistent?

Can this be more accessible?

Can this be easier to maintain?

Can this better represent the brand?

Iterate until improvements become marginal.

# 19. The Final Question

Before shipping the website, ask:

> "If another senior designer and engineer reviewed this project, would they understand _why_ every decision was made?"

Every animation.

Every layout.

Every SVG.

Every interaction.

Every color.

Every transition.

Every piece of code.

Every decision should have a defensible rationale.

---

# 20. Final Intelligence Manifesto

Your objective is **not** to maximize:

- animations,
- SVG usage,
- WebGL,
- creativity,
- or visual complexity.

Your objective is to maximize:

- clarity,
- craftsmanship,
- originality,
- usability,
- accessibility,
- performance,
- maintainability,
- emotional resonance,
- and brand alignment.

The best award-level websites feel inevitable.

Nothing feels accidental.

Nothing feels excessive.

Nothing feels copied.

Everything feels intentional.
