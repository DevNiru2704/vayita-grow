# MASTER AWWWARDS WEBSITE PROMPT

## Part 8 - Development Workflow, Acceptance Criteria & Final Master Instructions

> This final section converts the previous seven parts into an executable workflow. It defines how the AI should approach every project from discovery to delivery, ensuring consistency, adaptability, and production-quality results.

---

# 1. Master Directive

This document is **not** a UI template.

It is **not** an animation checklist.

It is **not** a list of components that must be included.

It is an intelligent design framework.

Your objective is to create the **best possible website for the given project**, using only the techniques that genuinely improve the final experience.

Every decision should be intentional.

---

# 2. Overall Workflow

Always follow this order.

```
Understand the Project
        ↓
Analyze Brand
        ↓
Determine Users
        ↓
Determine Business Goals
        ↓
Extract Design System
        ↓
Determine Motion Language
        ↓
Choose Technical Stack
        ↓
Plan Information Architecture
        ↓
Design Components
        ↓
Design Storytelling
        ↓
Prototype Interactions
        ↓
Build
        ↓
Optimize
        ↓
Accessibility Review
        ↓
Performance Review
        ↓
Consistency Review
        ↓
Final Polish
        ↓
Production Ready
```

Never skip early planning.

Good engineering begins long before writing code.

# 23. Final Master Prompt

When generating the final website, follow this mindset:

> Build the website that this project deserves-not the website that showcases the most techniques.

Design should emerge from understanding.

Motion should emerge from storytelling.

Engineering should emerge from maintainability.

Creativity should emerge from the brand.

Performance should never be sacrificed.

Accessibility should never be optional.

Every component should feel intentional.

Every interaction should have purpose.

Every line of code should be maintainable.

Every animation should strengthen the experience.

The final result should feel cohesive, timeless, and uniquely suited to the project.

---

# 24. Universal Principle

If at any point this master prompt appears to conflict with good design judgment:

Choose good design judgment.

If a listed technique weakens the experience:

Do not use it.

If removing an animation improves clarity:

Remove it.

If simplifying a layout improves usability:

Simplify it.

If restraint creates a stronger product:

Choose restraint.

Professional design is defined as much by what is omitted as by what is included.

---

# End of Master Prompt

## Using This Prompt

For every new project, append your project-specific brief **after** this master prompt.

Example:

```text
MASTER PROMPT (Parts 1-8)

↓

Project Brief

↓

Brand Guidelines

↓

Content

↓

Functional Requirements

↓

Technical Constraints

↓

Expected Deliverables
```

The AI should always:

1. Read the master prompt.
2. Read the project-specific prompt.
3. Extract the project's unique design system.
4. Decide which principles and techniques from the master prompt are appropriate.
5. Ignore everything that does not improve the project.
6. Produce a bespoke, original, production-ready website aligned with the project's goals.

> **Final Principle:**  
> The success of the website is measured not by how many advanced techniques it contains, but by how naturally design, motion, engineering, storytelling, and usability come together into a single, memorable experience.
